# LOCKD — Firestore & Storage Data Model

This document defines the Firestore collection layout, document shapes, Storage paths, and required composite indexes for the LOCKD MVP.

**Field names track the iOS Codable structs exactly** — `LockDUser.swift`, `Post.swift`, `StudySession.swift`. Anything listed below but not declared on those structs is a server-internal field; Swift's `Codable` decoder ignores it safely.

Conventions:
- All timestamps are Firestore `Timestamp` (server-generated where possible).
- All IDs are auto-generated by Firestore unless otherwise noted (`{uid}` means the Firebase Auth UID).
- Counters (reactionCount, commentCount, postCount, streak, etc.) are **denormalized** — only written from Cloud Functions.
- Client code must never write to fields marked `(server-only)`.

---

## 1. Collections

### `users/{uid}`
One doc per registered user. The doc ID is the Firebase Auth UID. Fields map 1:1 onto `LockDUser.swift`; the trailing rows are server-internal.

| Field | Type | Writable by | Notes |
|---|---|---|---|
| displayName | string | owner | 1–30 chars |
| photoURL | string\|null | owner | from Storage or SSO |
| username | string | server-only | set via `claimUsername` callable; unique |
| xp | number | server-only | lifetime XP. 1 XP per minute studied. |
| currentStreak | number | server-only | consecutive study-days (UTC) |
| timesLockedIn | number | server-only | total number of scored sessions |
| minutesPerWeek | number | server-only | rolls over every Monday 00:00 UTC |
| previousWeekPoints | number | server-only | snapshot of `xp` at week rollover |
| lastActiveAt | Timestamp | owner | bumped on app open; server also touches it |
| createdAt | Timestamp | server-only | set by `onUserCreated` |
| updatedAt | Timestamp | server-only | server bumps whenever it writes |
| *(internal)* longestStreak | number | server-only | not read by iOS today |
| *(internal)* lastStudyDay | Timestamp\|null | server-only | UTC day of last scored session |
| *(internal)* postCount | number | server-only | bumped on post create/delete |

### `usernames/{username}`
Lookup doc to enforce uniqueness. `username` is lowercased. Field: `{ uid: string }`. Written only by `claimUsername`.

### `posts/{postId}`
One doc per shared study clip. `postId` is auto-generated. Fields map 1:1 onto `Post.swift`; the trailing rows are server-maintained.

| Field | Type | Writable by | Notes |
|---|---|---|---|
| authorId | string | server-only (callable) | == the caller's uid |
| authorName | string | server-only | denormalized from `users.displayName` |
| authorPhotoURL | string\|null | server-only | denormalized |
| videoURL | string | server-only (callable) | Storage URL |
| thumbnailURL | string\|null | server-only (callable) |  |
| caption | string | author | ≤ 280 chars, editable after create |
| createdAt | Timestamp | server-only | serverTimestamp on create |
| *(internal)* sessionId | string | server-only | links back to the session |
| *(internal)* durationMinutes | number | server-only | copied from the clamped session |
| *(internal)* reactionCount | number | server-only | incremented by trigger |
| *(internal)* commentCount | number | server-only | incremented by trigger |

> Posts can only be created via the `createPostFromSession` HTTPS callable. Direct client writes to `posts/{postId}` are blocked by the rules.

### `posts/{postId}/comments/{commentId}`
| Field | Type | Writable by |
|---|---|---|
| authorId | string | owner (on create) |
| authorName | string | owner |
| text | string | owner (≤ 500 chars) |
| createdAt | Timestamp | owner (serverTimestamp) |

### `posts/{postId}/reactions/{uid}`
Doc-per-user so the client can check "has this user reacted?" with a single `get`. `uid` is the reacting user's UID.

| Field | Type |
|---|---|
| type | string (`"fire"`, `"clap"`, `"brain"`) |
| createdAt | Timestamp |

### `sessions/{sessionId}`
The authoritative study-session record. Field names match `StudySession.swift`. `onSessionCreated` clamps the duration, awards XP, and updates the user's streak + counters.

| Field | Type | Writable by |
|---|---|---|
| userId | string | owner (on create) |
| taskType | string | owner (e.g. "Studying") |
| description | string\|null | owner |
| durationMinutes | number | owner (on create; immutable after) |
| createdAt | Timestamp | owner (serverTimestamp) |
| *(internal)* verifiedDurationMinutes | number | server-only | clamped duration actually credited |
| *(internal)* xpAwarded | number | server-only |
| *(internal)* postId | string\|null | server-only | set when a post is published from this session |

### `leaderboards/weekly/{weekId}/entries/{userId}`
Pre-computed weekly leaderboard. `weekId` is ISO week, e.g. `2026-W16`. Refreshed every 15 minutes by `refreshLeaderboards`.

| Field | Type |
|---|---|
| userId | string |
| displayName | string |
| photoURL | string\|null |
| score | number (minutes studied this week) |
| rank | number |
| updatedAt | Timestamp |

### `leaderboards/allTime/entries/{userId}`
Same shape; `score` is lifetime `xp`.

### `leaderboards/weeklyIndex/{weekId}`
Flat index of which weekly snapshots exist. Used by `gcOldWeeklySnapshots` to prune history older than 12 weeks.

## 2. Storage layout

```
videos/{uid}/{sessionId}.mp4          # the uploaded timelapse
thumbnails/{uid}/{sessionId}.jpg      # poster frame
avatars/{uid}.jpg                     # profile picture
```

Rules of thumb:
- Bucket path always starts with the owning `uid` so rules are trivial.
- Never store raw (un-timelapsed) video — compress client-side with `AVAssetExportSession` before upload.
- Target < 25 MB per video for MVP. Reject uploads > 100 MB.

---

## 3. Required composite indexes

Declared in `firestore.indexes.json`:

| Collection | Fields |
|---|---|
| posts | `authorId` ASC, `createdAt` DESC (profile feed) |
| posts | `createdAt` DESC (global feed — auto) |
| sessions | `userId` ASC, `createdAt` DESC (study history) |
| users | `xp` DESC (auto, leaderboard scan) |
| users | `minutesPerWeek` DESC (auto, weekly leaderboard scan) |
| leaderboards/weekly/{weekId}/entries | `rank` ASC (auto) |

---

## 4. Query patterns the app needs

| Screen | Query |
|---|---|
| Global feed | `posts` orderBy `createdAt` desc, limit 20, cursor-paginated |
| Profile feed | `posts` where `authorId == uid` orderBy `createdAt` desc |
| Comments list | `posts/{postId}/comments` orderBy `createdAt` asc |
| Did-I-react | `get posts/{postId}/reactions/{myUid}` |
| Weekly leaderboard | `leaderboards/weekly/{thisWeek}/entries` orderBy `rank` asc, limit 50 |
| All-time leaderboard | `leaderboards/allTime/entries` orderBy `rank` asc, limit 50 |
| Study history | `sessions` where `userId == uid` orderBy `createdAt` desc |

---

## 5. Trust boundaries (what the client is *not* allowed to set)

Anything that affects rank, XP, or streaks is server-computed. The security rules block:

- `users.xp`, `users.currentStreak`, `users.longestStreak`, `users.lastStudyDay`
- `users.timesLockedIn`, `users.minutesPerWeek`, `users.previousWeekPoints`
- `users.postCount`, `users.createdAt`, `users.updatedAt`
- `sessions.verifiedDurationMinutes`, `sessions.xpAwarded`, `sessions.postId`
- direct creates on `posts/{postId}` (must use `createPostFromSession`)
- `posts.reactionCount`, `posts.commentCount`, `posts.durationMinutes`, `posts.sessionId`
- all `leaderboards/**` writes

The security rules in `firestore.rules` enforce this; see `BACKEND_SETUP.md` for deployment.
